## Chapter Structure

Each chapter follows a consistent format:

1. **Overview** - What you'll learn and why it matters
2. **Core Concepts** - Fundamental explanations
3. **Forms** - Different ways to use the feature
4. **Common Gotchas** - Mistakes to avoid
5. **Quick Decision Guide** - When to use what
6. **Code Examples** - Real-world patterns with ❌ bad and ✅ good practices
7. **Summary** - Key takeaways

## Code Conventions

Throughout this book:
- **ES6+ syntax** is used by default
- **const/let** preferred over var
- **Strict equality (===)** over loose equality
- **Async/await** over callbacks
- **Module syntax (import/export)** in examples
