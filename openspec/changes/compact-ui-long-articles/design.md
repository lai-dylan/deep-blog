## Context

博客UI当前采用宽松设计，在深度长文场景下导致阅读效率低下。需要在保持极简风格的前提下，通过紧凑布局提升长文阅读体验。

## Goals / Non-Goals

**Goals:**
- 全局间距减少20-30%，提升信息密度
- 文章列表每屏显示更多条目
- 长文章阅读减少滚动次数
- 添加目录导航支持快速定位
- 保持极简设计风格不变

**Non-Goals:**
- 不改变配色方案
- 不添加复杂交互
- 不修改功能逻辑
- 不调整字体家族

## Decisions

### 1. 间距调整方案

**Decision**: 采用 CSS 变量集中控制，逐步递减策略

**具体调整**:
```css
/* 当前 -> 调整后 */
--space-xs: 0.25rem -> 0.25rem  (保持不变)
--space-sm: 0.5rem  -> 0.375rem  (减小)
--space-md: 1rem    -> 0.75rem   (减小)
--space-lg: 1.5rem  -> 1rem      (减小)
--space-xl: 2rem    -> 1.5rem    (减小)
--space-2xl: 3rem   -> 2rem      (减小)
```

**Rationale**:
- CSS变量便于统一调整和维护
- 保持比例关系，视觉层次不乱
- 长文场景下紧凑但不拥挤

### 2. 行高调整

**Decision**: 从 1.75 调整为 1.65

**Rationale**:
- 1.75 适合短文本，1.65 更适合长文阅读
- 中文内容 1.6-1.8 是舒适区间
- 紧凑但不过密，保护阅读体验

### 3. 文章列表密度

**Decision**: 减少 ArticleCard 间距，从 space-y-8 改为 space-y-4

**影响**:
- 每屏从显示3篇增加到5-6篇
- 标题和摘要间距相应减小
- 标签间距 tighter

### 4. 目录导航实现

**Decision**: 右侧悬浮固定 TOC，支持自动高亮当前章节

**Implementation**:
- 通过 IntersectionObserver 监听标题进入视口
- 文章 H2/H3 自动生成目录
- 固定定位在右侧 aside 区域
- 移动端收起为可展开按钮

**位置**: `app/posts/[slug]/page.tsx` 中添加 TOC 组件

### 5. 响应式策略

**Decision**: 桌面端紧凑，移动端保持舒适

**Breakpoints**:
- < 768px: 保持当前宽松间距，触控友好
- >= 768px: 应用紧凑布局，提升密度

## Risks / Trade-offs

| Risk | Mitigation |
|------|-----------|
| 过于紧凑影响可读性 | 保留最小行高 1.65，不压缩到 1.5 以下 |
| 用户不适应新密度 | 保持设计一致性，只是微调间距 |
| 移动端体验下降 | 移动端保持宽松，仅桌面端紧凑 |
| TOC 遮挡内容 | TOC 宽度控制在 200px，可收起 |

## Migration Plan

1. 更新 CSS 变量和间距类名
2. 调整各组件 margin/padding
3. 添加 TOC 组件到文章页
4. 测试不同长度文章的显示效果
5. 验证移动端保持正常
